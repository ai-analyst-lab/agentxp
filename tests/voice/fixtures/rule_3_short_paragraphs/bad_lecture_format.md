To understand this data, you need to first appreciate that the checkout funnel is a multi-stage process where each stage produces its own event, and the events in this table are a denormalized view of those stages with one row per user-session combination, which means some users will appear multiple times if they had multiple sessions during the observation window, and the implications of that for any aggregation you want to do are significant because you cannot treat rows as users unless you first deduplicate by user_id, and similarly you cannot treat rows as conversions because a single user can have multiple sessions where they progressed partway through the funnel before finally converting in a later session, all of which complicates the simple denominator question of how many people experienced this experience and how many of them converted at each stage.
